数组

扩容机制:在原本容量cap只有10的情况下，我们向数组里面添加11个元素，此时Size>cap，则会触发扩容机制。

  1. 以一定的规则创建出一个新的容量>cap的新数组。
  2. 将原数组数据按原来的位置复制到新数组中去。
  3. 最后再将之前放不下的元素塞到新数组中去。
  • Size: 数组内元素个数
  • Cap: 数组最大实际容量

addLast方法

向数组的元素尾部添加元素addLast方法。

思路：将我们要添加的元素作为参数传递到方法之中，并将该元素赋值给array[Size],每成功完成一次赋值Size++

add方法

向数组的任意位置添加单个元素add方法。

思路：1.传递一个需要添加的位置索引。2.将该索引后的所有元素的索引位置全部往后挪一位。3.完成元素的添加。4.Size++

思考：index能随便传吗？

1.index > Size ——>可进行添加

2.index<Size ——>可进行添加

3.index=Size ——>可添加，不过此时相当于直接往数组尾部进行添加执行addLast方法

  • 如果有5个元素，如果要向1号索引位置添加元素，则从2号索引开始往后的所有元素也就是5-1=4位元素全部往后挪一位，那么1号索引位置就空下来了。
  • System.arraycopy（）四个参数含义
    • array：要进行拷贝的数组。
    • index：要进行拷贝的元素索引。
    • index + 1: 要从哪个索引位置开始拷贝。
    • size - index: 从要进行拷贝的索引位置起往后一共需要拷贝多少个元素。

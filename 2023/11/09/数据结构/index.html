数组

扩容机制:在原本容量cap只有10的情况下，我们向数组里面添加11个元素，此时Size>cap，则会触发扩容机制。

  1. 以一定的规则创建出一个新的容量>cap的新数组。
  2. 将原数组数据按原来的位置复制到新数组中去。
  3. 最后再将之前放不下的元素塞到新数组中去。
  • Size: 数组内元素个数
  • Cap: 数组最大实际容量

addLast方法

向数组的元素尾部添加元素addLast方法。

思路：将我们要添加的元素作为参数传递到方法之中，并将该元素赋值给array[Size],每成功完成一次赋值Size++

add方法

向数组的任意位置添加单个元素add方法。

思路：1.传递一个需要添加的位置索引。2.将该索引后的所有元素的索引位置全部往后挪一位。3.完成元素的添加。4.Size++

思考：index能随便传吗？

1.index > Size ——>可进行添加

2.index<Size ——>可进行添加

3.index=Size ——>可添加，不过此时相当于直接往数组尾部进行添加执行addLast方法

  • 如果有5个元素，如果要向1号索引位置添加元素，则从2号索引开始往后的所有元素也就是5-1=4位元素全部往后挪一位，那么1号索引位置就空下来了。
  • System.arraycopy（）四个参数含义
    • array：要进行拷贝的数组。
    • index：要进行拷贝的元素索引。
    • index + 1: 要从哪个索引位置开始拷贝。
    • size - index: 从要进行拷贝的索引位置起往后一共需要拷贝多少个元素。

带哨兵的单向链表

在单向链表的方法实现中，很多时候都要考虑该链表是否为空或者索引为0的的情况。

用一个不参与数据存储的特殊 Node 作为哨兵，它一般被称为哨兵或哑元，拥有哨兵节点的链表称为带头链表。

  • 具体存什么值无所谓，因为不会用到它的值

  • 遍历的节点需改为head.next 。哨兵的下一个节点。

  • 不用考虑索引为0、链表为空的情况。

之前的一些方法可以进行一些改动。

二叉树

二叉树是这么一种树状结构：每个节点最多有两个孩子，左孩子和右孩子

重要的二叉树结构

  • 完全二叉树（complete binary tree）是一种二叉树结构，除最后一层以外，每一层都必须填满，填充时要遵从先左后右

    • 堆就是一种完全二叉树

  • 平衡二叉树（balance binary tree）是一种二叉树结构，其中每个节点的左右子树高度相差不超过 1

  • 二叉搜索树 左孩子的值都比当前节点小，右孩子的值都比当前节点大

遍历

遍历也分为两种

  1. 广度优先遍历（Breadth-first order）：尽可能先访问距离根最近的节点，也称为层序遍历
  2. 深度优先遍历（Depth-first order）：对于二叉树，可以进一步分成三种（要深入到叶子节点）
    1. pre-order 前序遍历，对于每一棵子树，先访问该节点，然后是左子树，最后是右子树
    2. in-order 中序遍历，对于每一棵子树，先访问左子树，然后是该节点，最后是右子树
    3. post-order 后序遍历，对于每一棵子树，先访问左子树，然后是右子树，最后是该节点

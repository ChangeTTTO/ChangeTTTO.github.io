双向链表带哨兵

雏形结构。头节点就是哨兵，哨兵就是头

两个小块分别为头尾的上一个节点和下一个节点。

双向链表可以快速的找到尾节点，因为尾哨兵指向的就是尾节点

  • 链表属性

    • head：头部哨兵
    • tail: 尾部哨兵

  • Node类

    • prev：上一个节点指针
    • next： 下一个节点指针
    • value: 节点值

1.根据索引寻找节点findNode方法

会找到哨兵节点。

private仅供内部调用。

不会找到tail节点。

  • 因为索引0是真实添加的节点，而在索引0前有一个头哨兵，这个节点就是-1节点，也即是从-1开始遍历。
  • 因为尾节点本神旧不存储数据了，所以当p=tail的时候就应该停止循环。
  • 初始时p=head，每一次循环之后p=p.next

2.向索引位置插入insert节点（数据）

  • 先找到它的上一个节点。

  • 如果上一个节点找不到，抛异常。

  • 找到它的下一个节点

  • 创建一个新节点，新节点的prev指向上一个节点，新节点的next指向下一个节点。

  • 使上一个节点的next指向新节点。

  • 使下一个节点的prev指向新节点。

3.按索引删除元素remove

找到上一个和后一个元素，将其之间的关联断开

  1. 先拿到上一个元素
  2. 再拿到要删除的元素
  3. 再拿到后一个元素
  4. 将prev的next指针指向next
  5. 将下一个元素的prev指向上一个元素。

4.addLast向尾部添加数据

  • 先通过尾哨兵拿到最后一个节点

  • 创建新节点，它的prev指向最后一个节点

  • 新节点的next指向尾哨兵

  • 上一个节点的next指向新节点

  • 尾哨兵的prev指向新节点

5.removeLast删除最后一个节点

  1. 先通过尾哨兵拿到待删除的节点。

  2. 再通过待删除的节点拿到它的上一个节点prev。

  3. 使prev的next指向尾哨兵。

  4. 使尾哨兵的prev指向上一个节点。

  5. 如果链表中没有元素了，那么尾哨兵的上一个元素就是头哨兵，此时需要抛出异常。

双向环形链表-带哨兵

双向环形链表带一个哨兵。这时哨兵既作为头，也作为尾

基础框架

最初哨兵的prev和next都是自己指向自己的。

1.addFirst添加元素到首位

  1. 先找到哨兵
  2. 找到哨兵的下一个节点
  3. 创建一个新节点，让新节点的prev指向哨兵，next指向哨兵的下一个节点。
  4. 让哨兵的next指向新节点。
  5. 让下一个节点的prev指向新节点。

2.实现迭代器方法

3.addLast添加最后一个

  1. 先找到哨兵的下一个节点作为新节点的next
  2. 再找到哨兵作为新节点的prev
  3. 创建一个新节点。
  4. 使上一个节点的next指向新节点。
  5. 使下一个节点的prev指向新节点。

4.删除第一个

  1. 找到被删除的节点，也就是哨兵的下一个节点。
  2. 找到它的上一个节点，也就是哨兵。
  3. 找到它的下一个节点，也就是removed.next
  4. 让哨兵的next指向removed.next
  5. 让removed.next的prev指向哨兵
  • 如果要删除的节点是哨兵，抛出异常。

Kotlin Data Types

Kotlin 中最基本的数据类型是原始数据类型，所有其他数据类型都是引用类型，例如数组和字符串。Java 需要使用包装器 (java.lang.Integer) 来使原始数据类型表现得像对象，但 Kotlin 已经将所有数据类型都视为对象。

Kotlin 中有不同的数据类型：

  1. 整数数据类型 Integer Data type
  2. 浮点数据类型 Floating-point Data Type
  3. 布尔数据类型 Boolean Data Type
  4. 字符数据类型 Character Data Type

整数数据类型

这些数据类型包含整数值。

数据类型 位 最小值 最大值
byte 8位 -128 127
short 16位 -32768 32767
int 32位 -2147483648 2147483647
long 64位 -9223372036854775808 9223372036854775807

当初始化一个没有显式指定类型的变量时，编译器会自动推断为自 Int 起足以表示该值的最小类型。 如果不超过 Int 的表示范围，那么类型是 Int。 如果超过了，那么类型是 Long。 如需显式指定 Long 值，请给该值追加后缀 L。 显式指定类型会触发编译器检测该值是否超出指定类型的表示范围。

Kotlin when 表达式

在 Kotlin 中，when取代了Java 等其他语言的switch运算符。当满足某些条件时需要执行特定的代码块。when 表达式的参数与所有分支一一比较，直到找到匹配项。找到第一个匹配项后，它到达when块的末尾并执行when块旁边的代码。与 Java 或任何其他编程语言中的 switch case 不同，我们不需要在每个 case 末尾有一个 break 语句。在 Kotlin 中，when可以通过两种方式使用：

  • 当作为一个声明
  • 当作为表达式（计算出一个结果然后赋值给别人）

将when 与else 一起用作语句

when可以用作带或不带 else 分支的语句。如果用作语句，则将所有单独分支的值依次与参数进行比较，并执行条件匹配的相应分支。如果没有一个分支满足条件，那么它将执行 else 分支。

Kotlin continue关键字

在编程中使用循环时，有时需要跳过循环的当前迭代。在这种情况下，我们可以在程序中使用continue语句。基本上，continue用于针对特定条件重复循环。它会跳过以下语句并继续循环的下一次迭代。
Kotlin 中有两种类型的 continue
众所周知，无标签的 continue用于跳过最近的闭循环的迭代，而带标签的continue用于跳过所需的闭循环的迭代。可以借助inner@、outer@等标签。我们只需要在表达式前面写一个标签，然后使用 continue@abc 调用即可。

在 while 循环中使用带标签的 continue

带标签的continue用于在满足特定条件时跳过所需块的迭代，而不检查 while 循环中的条件。如果您使用标签outer@标记外层循环，并使用inner@标记内层循环，那么您可以在条件块中使用continue@outer轻松跳过特定条件。

Kotlin 中的引用相等性是通过**===运算符以及逆运算符！==检查的。仅当类型的两个实例都指向内存中的同一位置时，此相等性才返回 true。当用于在运行时转换为基元类型的类型时，===** 检查将转换为 == 检查，!== 检查将转换为 != 检查。

在 Kotlin 中，函数用于封装一个可以多次执行的行为。函数可以接受输入参数、返回值，并提供一种将复杂逻辑封装到可重用代码块中的方法。

函数是执行特殊任务的代码单元。在编程中，函数用于将代码分解为更小的模块，使程序更易于管理。

例如：如果我们必须计算两个数字的总和，那么定义一个**sum()**。

我们可以多次调用 sum(x, y) ，它将返回两个数字的和。因此，函数避免了代码的重复，使代码更具可重用性。在 Kotlin 中，有两种类型的函数 -

  • 标准库函数
  • 用户自定义函数

调用用户定义的函数

我们创建一个函数来分配特定任务。每当调用函数时，程序都会离开当前代码段并开始执行该函数。函数的流程控制-

  1. 程序到达包含函数调用的行。
  2. 当函数被调用时，控制权转移到该函数。
  3. 将函数的所有指令一一执行。
  4. 仅当函数到达右大括号或有任何 return 语句时，控制权才会转回。
  5. 函数返回的任何数据都用于代替原始代码行中的函数。

在 Kotlin 中使用函数的优点：

  1. 模块化：函数提供了一种模块化代码并将其分解为更小、更易于管理的部分的方法。这使您的代码更具可读性、可维护性并且更易于测试。
  2. 可重用性：可以从代码中的多个位置调用函数，从而可以轻松重用代码并避免重复。
  3. 提高可读性：函数提供了一种将复杂逻辑封装到可重用代码块中的方法，这可以使您的代码更具可读性和更容易理解。
  4. 改进的抽象：函数可用于抽象出复杂的逻辑，从而更容易理解一段代码的作用，而无需检查其实现细节。

在 Kotlin 中使用函数的缺点：

  1. 开销：函数会增加代码的大小并增加执行代码所需的内存量，特别是当您有许多函数时。
  2. 调试：如果函数内部有复杂的逻辑，尤其是有多个相互调用的函数时，函数会使调试变得更加困难。

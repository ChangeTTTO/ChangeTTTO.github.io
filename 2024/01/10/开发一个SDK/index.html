如何开发一个SDK

发表于2024-01-10|更新于2024-01-13
|阅读量:

开发一个简单易用的SDK的详细步骤

新建一个 spring boot 初始化项目

添加依赖，Lombok, Spring Configuration Processor

  • Spring Configuration Processor 的作用是自动生成代码提示

修改 pom 文件的版本号，并删除 build

删除原本自动创建的主类，新建一个类, 并添加需要用到的依赖

  • @configurationproperties注解这个properties注解它的作用是什么呢？它的作用就是能够读取我们这个。application的配置，然后读取到配置，把这个读到的配置呢设置到我们这里的属性中

  • @Bean生成一个客户端

  • 引入这个SDK并写好配置文件后，就会生成一个客户端对象。

在 resources 目录下新建 META-INF 目录，并创建 spring.factories 文件，并指定配置类的目录

org.springframework.boot.autoconfigure.EnableAutoConfiguration

install 打包构建在本地的仓库

复制 groupId, artifactId, version,可以在其他项目中添加为依赖

在另一个模块的配置文件当中就可以引用 starter 当中设置的属性

引入客户端调用接口

可以引入 starter 当中的类或者属性

关键点
1、确认自己的依赖

2、META-INF指定自己的配置注册类

3、引入Spring Configuration Processor

自动提示原理分析

总结

开发一个简单易用的SDK

理想情况：开发者只需要关心调用哪些接口、传递哪些参数，就跟调用自己写的代码一样简单。

开发starter的好处：开发者引入之后，可以直接在application.yml中写配置，自动创建客户端

springboot-boot-configuration-processor的作用就是自动生成配置的代码提示
